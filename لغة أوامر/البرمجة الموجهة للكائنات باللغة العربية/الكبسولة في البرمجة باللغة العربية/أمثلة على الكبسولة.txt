أكتب مباشرة الكود البرمجي المقترح بشكل صحيح وجملة صحيحة يمكن قرائتها وفهمها من قبل الجميع باللغة العربية الفصحى مع ذكر مثال لطريقة تطبيق هذا الأمر البرمجي 

لنبدأ بعرض مثال بسيط على الكبسولة باستخدام اللغة العربية الفصحى. دعنا نفترض أن لدينا كائن "سيارة" يجب تكبسله. 

``` 
إنشاء كائن سيارة {
     خاص سرعة؛

     دع تعيين_سرعة(قيمة) {
        إذا(قيمة >= 0) {
            سرعة = قيمة ;
        }
     }

     دع الحصول_على_سرعة() {
        بالعودة سرعة ;
     }

}

إنشاء كائن سيارة1 من سيارة ;
سيارة1.تعيين_سرعة(100) ;
طباعة (سيارة1.الحصول_على_سرعة()) ;
``` 

في الكود البرمجي الموجود أعلاه، لقد قمنا بتعريف الكائن "سيارة" الذي يحتوي على الخاصية الخاصة "سرعة". ثم أنشأنا طريقتين، واحدة لتعيين القيمة للسرعة "تعيين_سرعة"، والأخرى للحصول على القيمة "الحصول_على_سرعة". ثم أنشأنا كائن "سيارة1" من الكائن "سيارة" وعيّنّا السرعة لها ثم طبعنا القيمة.

من هذا الكود، نلاحظ أنه من الصعب تغيير قيم السرعة بشكل مباشر خارج الكائن سيارة بفضل الكبسولة، حيث يمكننا فقط تغييرها عن طريق استدعاء الطرق المتاحة لذلك. الطريقة "تعيين_سرعة" تضمن أن القيمة الجديدة للسرعة لا تكون أقل من الصفر، وهذا يناسب قاعدة العالم الحقيقي بأن سرعة السيارة لا يمكن أن تكون سالبة. هذا هو الجانب الحمايدي من الكبسولة.
