# التجاوز في الطرق بالبرمجة باللغة العربية

التجاوز في البرمجة يعني أن تكتب طريقة في الصف الفرعي تحمل نفس الاسم, الرجوع والمعلمات كالطريقة في الصف الأساسي. إذا نفذت الطريقة من الصف الأساسي, فالطريقة في الصف الفرعي ستتجاوز الطريقة في الصف الأساسي.

دعنا نشرح التجاوز في الطرق من خلال مثال سهل الفهم يتعلق بسيارتين من موديل وصفتا ولكن متفاوتتين في بعض الخصائص مثل المحرك.

أولاً، دعونا نعرف الصف الأساسي "سيارة"

```
صف سيارة
  نافذة_واجهة (دالة) {
     أرجع "تظليل باهت"
  }
  أنوع_المحركات(دالة){
     أرجع "4 أسطوانات"
  }
نهاية الصف
```
هنا قمنا بتعريف صف "سيارة" ذو طريقتين، واحدة مسماة "نافذة_واجهة" والأخرى "أنوع_المحركات". 

الآن دعونا نعرف الصف الفرعي "سيارة_رياضية"

```
صف سيارة_رياضية يرث سيارة
  نافذة_واجهة (دالة) {
     أرجع "تظليل داكن"
  }
  أنوع_المحركات(دالة){
     أرجع "8 أسطوانات"
  }
نهاية الصف
```
في الصف "سيارة_رياضية"، تم تجاوز الطرق "نافذة_واجهة" و "أنوع_المحركات"، وذلك بتغيير قيمها الافتراضية.

في النهاية، إذا كنا نتعامل مع كائن من صف "سيارة"، فستكون النتيجة كالتالي:

```
جديد سيارة;
=> نافذة_واجهة = "تظليل باهت", أنوع_المحركات = "4 أسطوانات"
```

وإذا كنا نتعامل مع كائن من صف "سيارة_رياضية"، فستكون النتيجة كالتالي:

```
جديد سيارة_رياضية;
=> نافذة_واجهة = "تظليل داكن", أنوع_المحركات = "8 أسطوانات"
```

هذا هو فهم التجاوز في الطرق باللغة البرمجة بالعربية الفصحى.
